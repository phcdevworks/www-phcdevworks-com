# BRAND.md

## Brand Purpose

PHCDevworks presents itself as a modern technical company with a restrained,
premium, systems-driven visual identity.

This website is the public company website for PHCDevworks. It may feature
Spectre prominently, but the visual system should support the broader
PHCDevworks brand and ecosystem rather than frame the site as a single-product
landing page.

## Brand Summary

The PHCDevworks website should feel:

- dark
- technical
- high-contrast
- typography-first
- premium
- minimal
- structured
- modern
- confident
- restrained

The site should not feel:

- playful
- pastel
- soft
- cute
- generic SaaS
- corporate-stock-photo driven
- illustration-heavy
- trend-driven
- gradient-led
- visually noisy

## Core Visual Identity

The identity is built on:

- black or near-black backgrounds
- white or off-white typography
- a primary accent color: `#EE005A` (Raspberry Red)
- a secondary deep color: `#012641` (Deep Space Blue)
- strong type hierarchy
- restrained surface layering
- subtle technical grid treatments
- selective use of accent bars and dividers
- limited, intentional glassmorphism

## Color System

### Primary Accent

Primary brand accent:

- `#EE005A` — Raspberry Red

This accent is locked unless explicitly changed by direct instruction.

### Secondary Color

Secondary brand color:

- `#012641` — Deep Space Blue

Use Deep Space Blue for:

- dark section backgrounds
- deep surface layering
- structural backdrop contrast
- supporting the primary accent without competing with it

### Base Palette Direction

Use a dark-first palette built around:

- black / near-black backgrounds
- `#012641` (Deep Space Blue) for deep structural surfaces
- white / off-white primary text
- neutral grays for borders, dividers, muted text, and subtle surface separation
- `#EE005A` for emphasis and interactive priority

### Color Rules

- never introduce blue as a casual or trend-driven choice — Deep Space Blue is approved only as a structural dark surface color
- never switch the site into a blue-dominant or blue-led palette
- do not rotate accent colors by page or section
- do not use multicolor gradients as branding
- use `#EE005A` selectively so it retains strength and perceived value
- use `#012641` as a dark structural surface — not as a bright or accent color

### Accent Usage

Use the accent color for:

- primary CTAs
- active states
- accent bars
- underline motifs
- focused highlight words in headlines
- key interactive emphasis
- small UI emphasis details

Do not use the accent color for:

- large paragraphs of body text
- entire sections unless carefully restrained
- gratuitous glow effects
- broad gradients

## Typography

Typography is one of the primary brand tools. The website should feel
typography-first, not image-first.

### Typography Direction

Use:

- one primary sans-serif family across the site
- bold, high-impact headings
- strong hierarchy
- concise supporting copy
- controlled line lengths
- clean modern letterforms
- technical rather than playful tone

Avoid:

- frequent font switching
- decorative display fonts
- novelty fonts
- serif-led brand presentation unless explicitly directed
- casual inconsistency between pages

### Typography Rules

- use one primary sans-serif family for the full website
- do not change the primary font without explicit instruction
- headings should carry visual identity through weight and scale
- supporting copy should remain concise and readable
- do not use decorative typography effects as a substitute for structure

## Layout System

The site should create impact through layout and hierarchy rather than imagery.

### Layout Principles

Use:

- strong alignment
- generous negative space
- clean section rhythm
- consistent content widths
- structured composition
- asymmetry where it strengthens visual balance
- clear scan paths

Avoid:

- cluttered sections
- template-like sameness
- over-stacked cards
- weak vertical rhythm
- decorative filler with no system role

## Backgrounds and Texture

Backgrounds should support the brand through restraint and structure.

### Approved Background Language

- black or near-black base surfaces
- subtle engineering or technical grid patterns
- faint structural linework
- minimal texture
- restrained depth

### Not Approved

- blue gradients
- aurora-style blobs
- multicolor glows
- decorative abstract art
- generic hero illustrations
- stock imagery backgrounds
- loud textures

## Surface Design

Surfaces should feel premium, dark, and layered with restraint.

### Surface Principles

Use:

- subtle contrast shifts
- faint borders
- low-opacity overlays
- controlled depth
- crisp readability
- consistent radii and structure

Avoid:

- shadow-heavy cards
- loud glossy surfaces
- overly bright panels
- noisy layered visual systems

## Glassmorphism

Glassmorphism is allowed, but it is not the core brand. It is a support
treatment.

### Approved Glassmorphism

Allowed:

- subtle translucent dark panels
- restrained blur on header, cards, or callout surfaces
- low-contrast layering
- faint borders
- modest depth that improves hierarchy

### Not Approved

- strong blur everywhere
- bright frosted glass blocks
- flashy futuristic effects
- candy-like translucent UI
- glass panels that weaken readability

### Glass Rule

Glassmorphism should support hierarchy, not define the brand by itself.

## Imagery Policy

This brand is intentionally image-light and may remain fully image-free for
major sections of the site.

### Approved Visual Content

Approved visual content includes:

- typography-led sections
- interface screenshots if needed later
- diagrams
- system maps
- product visuals tied to real PHCDevworks work
- iconography aligned with the system
- subtle structural graphics such as grids or dividers

### Not Approved

- stock photography
- fake team photos
- generic SaaS illustrations
- abstract filler images
- decorative image blocks used to hide weak layout or copy

### Hero Rule

Hero sections should not depend on photos or decorative imagery. They should
create impact through:

- typography
- spacing
- structure
- contrast
- accent detail
- subtle background system treatment

## Signature Motifs

The site may reuse a small set of signature motifs:

- accent bars and underline rules
- subtle technical grid backgrounds
- selective accent emphasis inside headings
- thin borders and structured dividers
- dark layered surfaces
- restrained glass panels

Use motifs consistently and sparingly.

## Interaction Style

Interaction should feel clean, quick, and controlled.

### Interaction Rules

- keep hover states subtle
- use accent color for active or priority states
- prefer motion-light transitions
- avoid bouncy or playful motion
- avoid exaggerated transforms
- avoid gimmick animations

## Voice and Copy Behavior

The brand voice should match the visual system.

Copy should feel:

- concise
- technical
- credible
- confident
- direct
- modern
- controlled

It should not feel:

- bloated
- hype-heavy
- vague
- cute
- cliché-driven
- generic

## Brand Non-Negotiables

1. never change the primary accent from `#EE005A` without explicit instruction
2. never change the secondary color from `#012641` without explicit instruction
3. never use `#012641` as an accent or interactive color — it is a structural dark surface only
4. never introduce any other blue outside the approved `#012641`
5. never casually change the primary font
6. never use stock photos, generic illustrations, or decorative hero images
7. never replace the dark technical visual language with generic SaaS styling
8. never let glassmorphism overpower clarity or hierarchy
9. never solve weak layout with decorative filler
10. never let the site become visually noisy

## Definition of On-Brand

A design is on-brand when it:

- feels like a modern technical flagship
- uses dark surfaces with disciplined contrast
- relies on bold typography and strong layout
- uses `#EE005A` sparingly and intentionally
- keeps visuals restrained and premium
- avoids generic SaaS styling
- avoids imagery-led marketing patterns
- feels system-driven, not trend-chasing
